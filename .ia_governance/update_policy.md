# Politique de mise à jour du GPS

## Mise à jour obligatoire

Mettre à jour `.ia_governance/project_gps.json` si :

- un nouveau dossier est ajouté
- un nouveau module est ajouté
- une nouvelle API importante est ajoutée
- un fichier important change de rôle
- un élément est archivé
- une nouvelle brique MVP est ajoutée

## Mise à jour recommandée

Mettre à jour le GPS si :

- une page UI importante est ajoutée
- un flux fonctionnel change
- un fichier devient critique pour le fonctionnement

## Mise à jour non obligatoire

Pas besoin de mise à jour immédiate si :

- petite correction de style
- texte ou label mineur
- refactor interne sans changement de rôle

## Principe

Le GPS doit rester :

- lisible
- synthétique
- fidèle au projet réel
- dépourvu de toute simulation ou "petite tricherie" technique.

Il ne doit pas devenir une copie brute du repo sans description.
