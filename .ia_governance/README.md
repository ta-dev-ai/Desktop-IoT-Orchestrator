# IA Governance

## But

Ce dossier sert à encadrer le travail des intelligences artificielles sur ce projet.

Il fournit :

- une carte structurée du repo
- les rôles des dossiers et fichiers importants
- une politique de mise à jour

## Ordre de lecture recommandé

1. `project_gps.json`
2. `update_policy.md`
3. la documentation métier dans `doc/` et `planning/`

## Règle importante

Si la structure, le rôle d’un module, ou un flux fonctionnel change :

- mettre à jour `project_gps.json`
- mettre à jour la documentation liée si nécessaire

## Objectif

Réduire les hallucinations et donner à toute IA une vue claire et maintenable du projet.
